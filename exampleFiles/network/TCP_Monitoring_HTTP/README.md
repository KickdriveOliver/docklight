# TCP Monitoring for HTTP

This example demonstrates Docklight TCP monitoring/pass-through mode by observing HTTP traffic between endpoints. It includes automation and screenshots for monitoring workflow validation.

## Requirements

- Docklight Scripting (network/HID/named-pipe examples are intended for Docklight Scripting)

## Files

| File | Description |
|------|-------------|
| TCP_Monitoring_HTTP.ptn | Project documentation (Notepad) |
| TCP_Monitoring_HTTP.ptp | Docklight project file |
| TCP_Monitoring_HTTP.pts | Docklight Script |
| TCP_Monitoring_HTTP_ptn.txt | Text export of documentation |
| TCP_Monitoring_HTTP_ptp.txt | Text export of project file |
| TCP_Monitoring_HTTP_pts.vbs | VBScript export of script |
| TCP_Monitoring_logfile_nic_com_asc.htm | Example asset |
| TCP_Monitoring_OpenThisLinkWithABrowser.URL | URL shortcut |
| TCP_Monitoring_screenshot_docklight_HTML_passThrough.png | Screenshot |
| TCP_Monitoring_screenshot_firefox_connects_to_docklight.png | Screenshot |

## Getting Started

1. Open `TCP_Monitoring_HTTP.pts` in Docklight Scripting.
2. Press `Shift+F5` to start the script. It loads the project via `DL.OpenProject` (target: `TCP_Monitoring_HTTP.ptp`).
