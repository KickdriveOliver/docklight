# CRC Reverse Engineering

This package helps identify unknown CRC algorithms from captured protocol traffic by iterating candidate parameters. It includes automation and screenshots for repeatable reverse-engineering workflows.

## Requirements

- Docklight Scripting (script-based workflow)
- Docklight can be used for project-only runs where no script is required

## Files

| File | Description |
|------|-------------|
| crc_reverse_engineering_crc_finder.ptn | Project documentation (Notepad) |
| crc_reverse_engineering_crc_finder.ptp | Docklight project file |
| crc_reverse_engineering_crc_finder_auto.pts | Docklight Script |
| crc_reverse_engineering_crc_finder_auto_pts.vbs | VBScript export of script |
| crc_reverse_engineering_crc_finder_ptn.txt | Text export of documentation |
| crc_reverse_engineering_crc_finder_ptp.txt | Text export of project file |
| docklight_crc_reverse_engijneering_example.png | Screenshot |

## Getting Started

1. Open `crc_reverse_engineering_crc_finder.ptp` in Docklight Scripting.
2. Press `F5`. The matching `crc_reverse_engineering_crc_finder_auto.pts` script is loaded automatically and started together with communication.
