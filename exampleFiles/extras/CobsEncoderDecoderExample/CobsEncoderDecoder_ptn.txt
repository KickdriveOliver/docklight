\par \cf1\lang1033\b\f3 Example project: Consistent Overhead Byte Stuffing (COBS)
\par \b0 O.Heggelbacher / 2022-01-26
\par 
\par This project demostrates how to use a Docklight script to COBS-encode Send Sequence data before transmission. And receive COBS packets back with a Receive Sequence and decode them using a corresponding decoder script. 
\par 
\par The script uses the \b _auto.pts \b0 feature so the encoder/decoder script is automatically started when you start communication / send data. See the Docklight User Manual at  https://docklight.de/manual/index.html?saving-and-loading-your-script.htm 
\par 
\par For easy testing without external devides, the project's communication settings are \b UDP:LOCALHOST:10001\b0 , which is basically a loopback where all TX data is received back immediately on the RX channel. 
\par 
\par The example Send Sequences are taken from the Wikipedia on COBS.  
\par \f4 https://en.wikipedia.org/wiki/Consistent_Overhead_Byte_Stuffing\cf2\f2 
\par \cf1\f4 
\par You  can test the project by clicking on one of the Example.. Send Sequences, and Docklight will send the COBS encoded packet and receive/decode it back, e.g Example05: 
\par 
\par \cf2\f2  ==========================
\par  Original data:
\par  11 00 00 00
\par  COBS-encoded transmission data:
\par 
\par 24.01.2022 12:14:34.623 [TX] - \cf3 02 11 01 01 01 00 \cf2 
\par 24.01.2022 12:14:34.623 [RX] - \cf4 02 11 01 01 01 00 \cf2  
\par  COBS decoded data: 
\par  \lang1031 11 00 00 00
